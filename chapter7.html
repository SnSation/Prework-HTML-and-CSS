Chapter 7: Forms

Form Controls

'control': A type of form

    Text:
  • Text Input: Used for a single line of text
  • Password Input: Same as text input, but hides characters
  • Text Area: Used for multiple lines of text
    Choices:
  • Radio Buttons: Select one option
  • Checkbox: Select one or more
  • Drop-Down Box: Select from a list
    Submission
  • Submit Button: Send data elsewhere
  • Image Button: Same as submit button, but uses an image
    Upload
  • File Upload: Allows file transfer from user to another location

Form Mechanics

  1. User inputs and submits data
  2. Each control's name is sent to the server with the input values
  3. Server uses data and may send a return

Form Structure

  • <form>: An element which collects user input and sends it somewhere else
  • 'action': An attribute of <form> which determines where data is sent
  • 'method': An attribute of <form> which determines how data is sent
    • 'get': A value of 'method' which adds the values from the form to the end of the URL specified in the 'action' attribute
        • 'get' Uses:
      • Search Boxes
      • Data Retreival
    • 'post': A value of 'method' which sends form values in HTTP headers
        • 'post' Uses:
      • File Upload
      • Large Amounts of Text
      • Sensitive Input
      • Database Modifications
    • 'get' is the default method if none is specified
  • 'id': The form's id value is used by scripts to identify a form and check its information

Input

    <input>: An element which determines the type of form control
  • 'name': An attribute of <input> which determines the name of the variable the input will be stored as
  • 'size': An attribute of <input> which determines the size of the text box
    • 'size' is a legacy attribute
    • CSS determines the size of modern text boxes
  • 'maxlength': An attribute of <input> which determines the maximum amount of characters a text field will allow
  • 'checked': An attribute of <input> which indicates if the <input> has been selected
    • Only 1 radio button should have a "checked" attribute and value
    • All selected checkbox buttons should have a "checked" attribute and value
  • 'placeholder':An attribute of <input> which will display its value as temporary text in a text field
  • 'type': An attribute of <input> which determines the type of value a form should accept
    • 'text': A value of the 'type' attribute which creates a single line text input
    • 'password': A value of the 'type' attribute similar to 'text', but masks characters on the user end
    • 'checked': An attribute of <input> which indicates if the <input> has been selected
      • Only 1 button should have '"checked" as the value of the 'checked' attribute, if any at all
    • 'radio':A value of the 'type' attribute which generates a button for the user to select
      • 'radio' only allows for 1 option to be selected
      • The 'name' value should be the same for all radio buttons in a given form
      • The 'value' value determines the value that will be stored in the 'name' variable if that options is selected
      • Text between 'radio' type tags will be displayed next to the button
        • The content of a radio button is ARBITRARY and IRRELEVANT to the server
        • Programatically, the only value that will be sent to the server is the value of the 'value' attribute
      • Once a button of the 'radio' type is selected, one button in that form must always be selected
    • 'checkbox': A value of 'type' which generates a checkbox
      • Checkboxes can be selected and deselected
      • Multiple checkboxes can be selected, simultaneously
    • 'file': A value of the 'type' attribute which allows the selection and submission of a file from the user
      • The 'file' value will generate a text box and a browse button
    • 'submit': A value of the 'type' attribute which sends input in the form to the location specified in the 'action' attribute
      • The value of the 'value' attribute determines the value displayed in the button
    • 'image': A value of the 'type' attribute which generates a submit button with the specified image and attributes
    • 'hidden': A value of the 'type' attribute which hides a form value from the user
    • 'date': A value of the 'type' attribute which generates a date input field
      • Older browsers do NOT recognize the 'date' type
    • 'email': A value of the 'type' attribute which generates an email submission form
      • Input that does not fill the requirements of an email address will NOT be validated
      • The 'email' type requires the browser to support HTML5 form validation
    • 'url': A value of the 'type' attribute which generates a url submission form
      • Input that does not fill the requirements of a URL will not be validated
      • The 'url' type requires the browser to support HTML5 form validation
    • 'search': A value of the 'type' attribute which generates a search submission form
      • Older browsers will treat 'search' type controls as a 'text' type controls
    <textarea>: An element used in a <form> which allows for multiple lines of text
  • Any text between the <textarea> tags will be entered in the form when it loads
    <select>: An element in <form> which generates a drop down list for the user to select from
  • <option>: An element of <select> which generates a list item for <select>
    • 'selected': An attribute of <option> which indicates if the <option> has been selected
  • 'size': An attribute of <select> which determines the number of <option> shown at once in <select>
    • 'multiple': An attrubute of <select> which allows users to select multiple <option> at once
    <button>: An element which indicates its contents are a button
  • <button> allows for greater control of the content and appearance of a button
    <label>: An element in <form> which makes its contents accessible to vision-impaired users
  • <label> can be used 2 ways:
    • <label> can be wrapped around the control (<input>) to add a text label associated with the control
    • <label> can be kept separate from the control and associated with it by using the 'id' and 'for' attributes
    • 'for': An attribute of <label> which associates it with any control with the same value for its 'id' attribute

Grouping Form Elements

    <fieldset>: An element which groups related controls together
  • <fieldset> is wrapped around all related controls
  • Most browsers will wrap elements inside <fieldset> tags with an box
  • <legend>: An element containing a caption for a <fieldset>
    • <legend> is written directly after the opening <fieldset>

Form Validation

Form Validation: A message indicating a form control has not been filled in correctly

Form Validation is usually done with JavaScript
